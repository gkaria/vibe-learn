# Spec 0001: Obsidian Integration

**Status:** Proposed
**Target version:** 0.5.0
**Date:** 2026-04-10

---

## Summary

Add bidirectional Obsidian integration to vibe-learn. Users can write session learnings to an Obsidian vault as tagged, frontmatter-rich notes, and recall past learnings by searching the vault.

## Motivation

vibe-learn captures what Claude builds during a session, but all output stays ephemeral — in-context or in `.vibe-learn/`. Users who maintain an Obsidian vault as a knowledge base have no way to persist their session learnings into it. This integration closes that loop: write what you learned, recall it later.

---

## Feature Design

### New Arguments

| Command | Direction | Behavior |
|---------|-----------|----------|
| `/learn obsidian` | Write | Generate learn summary, save to vault as a note |
| `/learn obsidian <question>` | Write | Answer question, save answer to vault as a note |
| `/learn obsidian:recall <topic>` | Read | Search vault for notes matching topic, synthesize cross-session learnings |
| `/digest obsidian` | Write | Generate full session digest, save to vault as a note |
| `/digest obsidian:recall` | Read+Write | Search vault for previous sessions on the same project, then generate digest enriched with a "Connections to Previous Work" section |

### Configuration

**File location:** `.vibe-learn/obsidian.json` (project-level) or `~/.vibe-learn/obsidian.json` (global fallback).

If no config exists when an obsidian command is invoked, Claude asks the user for their vault path and offers to save the config.

**Schema:**

```json
{
  "vault_path": "/Users/me/ObsidianVault",
  "subfolder": "Development/Sessions",
  "tags": ["vibe-learn"],
  "link_style": "wikilink",
  "include_project_tag": true,
  "note_naming": "{date}-{project}"
}
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `vault_path` | string | `""` (must be set) | Absolute path to the Obsidian vault root |
| `subfolder` | string | `"Development/Sessions"` | Folder within the vault for vibe-learn notes |
| `tags` | string[] | `["vibe-learn"]` | Tags added to every note's frontmatter |
| `link_style` | string | `"wikilink"` | `"wikilink"` for `[[links]]`, `"markdown"` for `[text](path)` |
| `include_project_tag` | bool | `true` | Auto-add project directory name as a tag |
| `note_naming` | string | `"{date}-{project}"` | Note filename template |

### Note Output Format

```markdown
---
date: 2026-04-10
created: 2026-04-10T14:32:00Z
project: my-app
tags:
  - vibe-learn
  - my-app
type: session-digest
---

# Session Digest — 2026-04-10

## What Was Built
2-4 sentence summary.

## Key Decisions
- Bullet points explaining choices and rationale.

## Patterns Used
- Brief bullets of techniques and concepts.

## Things to Study
- [ ] topic 1
- [ ] topic 2

## Session Activity
- Files created: 3
- Files modified: 5
- Commands run: 12

---
*Generated by [vibe-learn](https://github.com/gkaria/vibe-learn)*
```

**Frontmatter fields:**

| Field | Type | Purpose |
|-------|------|---------|
| `date` | date | Note date (YYYY-MM-DD), for Obsidian sorting |
| `created` | datetime | ISO 8601 timestamp of when note was generated |
| `project` | string | Directory name of the project |
| `tags` | list | Obsidian tags (from config + project name) |
| `type` | string | `"session-digest"` or `"learn"` — enables Dataview filtering |

**File naming:** `<YYYY-MM-DD>-<project>.md` for digests, `<YYYY-MM-DD>-<project>-learn.md` for learn notes. Numeric suffix appended on collision (e.g., `-2.md`).

### `/learn obsidian:recall` Behavior

When the user runs `/learn obsidian:recall <topic>`:

1. Load Obsidian config
2. Search `<vault_path>/<subfolder>/` for markdown files matching the topic (via Grep across titles, headings, body, tags)
3. Read up to ~5 most relevant matching notes
4. Synthesize a summary covering:
   - Which sessions touched this topic and when
   - Key decisions and patterns related to it
   - Any unchecked "Things to Study" items on this topic
   - Connections between sessions
5. Present summary to user — **no file is written**

### `/digest obsidian:recall` Behavior

When the user runs `/digest obsidian:recall`:

1. Load Obsidian config
2. Search `<vault_path>/<subfolder>/` for previous session notes from the **same project** (match on `project` frontmatter field or project name in filename)
3. Read the most recent matching notes (up to ~5)
4. Generate the full digest as normal, but enrich it with a **"Connections to Previous Work"** section that includes:
   - How this session builds on or continues previous work
   - Recurring patterns across sessions
   - Any unchecked "Things to Study" items from past sessions that are relevant to what was just built
   - Progression of understanding (e.g., "You first set up JWT auth on March 2nd, this session extends it with refresh tokens")
5. Write the enriched digest to the vault (same write behavior as `/digest obsidian`)
